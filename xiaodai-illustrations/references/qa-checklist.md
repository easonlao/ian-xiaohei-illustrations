# QA Checklist

## 必过项

- 画幅符合当前场景要求。
- 画幅是本次场景判断的结果，不是被默认写死的陈词滥调。
- 背景是干净白底。
- 有小呆。
- 小呆承担核心动作，不只是装饰。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 小呆外形稳定，没有因为换场景而被重新设计。
- 小呆没有长出独立躯干；整个红色本体仍然是单体火焰剪影。
- 两个白点眼睛清晰可见，没有丢失。
- 橙色只用于主路径或箭头。
- 红色只用于小呆本体、以及重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有 "常见坑 / Workflow / 系统架构图 / 路线图" 等标题。
- 小呆不像纯扁平红色剪影，或像可爱卡通、吉祥物。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 只是把旧案例原样照搬，几乎没有根据当前内容做调整。
- 明明用户没指定画幅，却直接滑回“默认 16:9 正文配图”而没有解释这次为什么这样判断。
- 同主题跨场景时，要么完全像复制粘贴，要么直接换成了另一个完全不同的隐喻。
- 小呆一会儿像尖火焰，一会儿像圆 mascot，一会儿像水滴人，IP 漂移明显。
- 小呆长出了独立躯干、胸腔、腹部或脖子。
- 红色本体被画成上下两段，读起来像头身分离或葫芦形身体。
- 小呆变成了无脸火焰，或眼睛缺失/弱到几乎看不见。
- 竖屏画面只是变高了，但主体过小、过正、过僵，没有真正的竖向张力。
- `article_inline` 画成正中小图标，小呆被几圈圆线围住，缺少横向解释感和段落呼吸区。

## 迭代方法

- 太普通：让小呆成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 serene、focused、not cute、not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和小呆动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。
- IP 漂：提高参考图优先级，收紧角色描述，减少会诱发重新设计的动作和道具。
- IP 还是漂：先降低动作复杂度和道具数量，不要一边角色不稳，一边还追求更复杂张力。
- 角色越界：把姿态先拉回更安全的单体火焰动作，不要继续尝试坐姿、蹲姿、蜷缩姿这类高风险动作。
- 竖屏太僵：优先改构图重心和方向冲突，不要只换成更长的比例。
- `article_inline` 太像贴纸：先把主体移出正中，明确一侧留白，再补一个横向外部锚点。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。
